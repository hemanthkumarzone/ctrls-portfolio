What are category views

One cost dataset. Every lens
your organisation needs.

Your cloud bill is a list of line items. Your AI cost reality is a set of business questions: Which models are driving spend growth? Which teams are over budget? How much are we spending on waste versus productive work? What does our GPU fleet cost per inference?

Category Views is the layer that bridges these two worlds. It maps every billing line item — every API call, every node-hour, every GB of storage — to one or more business-meaningful categories. Then it lets every stakeholder filter, slice, and drill into exactly the view they need, without requiring a data engineer to build a custom report.

Finance sees cost by category over time. Engineering sees cost by workload and namespace. FinOps sees waste by category and team. The CEO sees cost-as-percentage-of-revenue by AI product. All from the same underlying category model.

Category Views don't replace your billing dashboard — they sit on top of it, translating raw cloud costs into the language your organisation actually uses.

Category 01 — AI Models & APIs
AI Models & APIs
Every cost generated by calling, hosting, or licensing AI models — from third-party API tokens to self-hosted model serving infrastructure. Typically the largest and fastest-growing category for AI teams.
Token-level API costs — input tokens, output tokens, and cached tokens per model and per endpoint, attributed to team and product feature.
Model hosting costs — inference server compute, GPU allocation, and serving framework overhead for self-hosted or fine-tuned models.
Licensing and subscription fees — per-model SaaS subscriptions, enterprise API agreements, and usage-based licensing charges.
Fine-tuning API costs — the cost of customising foundation models, separated from standard inference costs for clear ROI visibility.
Model evaluation and testing — costs incurred by LLM evaluation pipelines, A/B testing infrastructure, and quality benchmarking runs.

Cross-category intelligence

Insights that only appear when
categories talk to each other

The most valuable findings in AI FinOps come from correlating signals across categories — not from looking at each in isolation.

🔗

Model cost vs infrastructure cost correlation

When model API costs rise but compute stays flat — you're getting less compute efficiency per dollar of model spend. When both rise proportionally, it's healthy scale.

Models ↔ Compute
⚡

GPU cost vs model API cost split

A high GPU-to-API ratio signals heavy self-hosting. A high API-to-GPU ratio signals heavy third-party dependency. Optimal split depends on scale — and changes as you grow.

GPU ↔ Models
💧

Data pipeline cost vs model quality

Expensive data pipelines should correlate with model performance improvements. When data cost grows but downstream model metrics plateau, the pipeline investment isn't paying off.

Data ↔ Models
📦

Storage growth vs waste accumulation rate

When storage cost grows faster than active workload count, artefacts are accumulating faster than they're being cleaned up. Category view correlation surfaces this before it compounds.

Storage ↔ Waste
🔀

MLOps cost vs team productivity

Rising MLOps tool costs should correlate with faster model deployment cycles and more experiments. If tooling cost grows without a corresponding velocity improvement, it's overhead not investment.

MLOps ↔ Compute
🌐

Network cost vs inference volume ratio

Egress cost should scale linearly with inference volume. A super-linear relationship signals response caching opportunities or unnecessarily verbose model outputs adding avoidable transfer costs.

Network ↔ Models

Platform integration

Category Views connect every
pillar to the same cost language

Every platform capability produces and consumes category data — creating a unified cost vocabulary across your entire AI FinOps programme.

PillarHow category views integrateBusiness outcome
📡 Monitoring Every monitoring signal — API costs, GPU telemetry, storage IO, network egress — is automatically categorised as data arrives. The monitoring dashboard surfaces costs in category view by default, so teams see spend in their own vocabulary rather than raw billing line items. Teams understand their cost data immediately without requiring a finance translation step. Every engineer and product manager can read their relevant category view without training.
⚙️ Optimization Right-sizing recommendations, model routing suggestions, and reserved capacity guidance are all scoped and ranked within their category — so the optimization engine can show "the top 5 compute category savings opportunities" or "the highest-ROI model category changes" without mixing recommendations across dimensions. Optimisation actions are presented within the business context they affect — making approvals faster and more intuitive for the teams who own each category.
🔀 Orchestration Workload scheduling decisions reference category cost data — routing jobs to the most cost-efficient compute option within the relevant infrastructure or GPU category. Category-level budget guardrails inform the orchestration engine's routing decisions in real time. Orchestration optimises within category budget constraints automatically — a GPU category budget cap triggers spot routing before the cap is hit, not after it's exceeded.
🗑️ Waste elimination The Identified Waste category is a direct output of waste elimination analysis — every idle resource, stale artefact, and over-provisioned workload surfaces in the waste category view with its elimination status, projected saving, and ownership information. Savings realised by waste elimination update the category in real time. The waste category becomes a living measure of FinOps programme effectiveness — growing when new waste is found, shrinking as actions are taken. Executives can track AI FinOps ROI from a single category trend line.
